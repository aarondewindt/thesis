		\section{Simulation results}
		I managed to get a tabular Q-learning RL agent to control the pole. The results shown below are for an agent with 1681 different states (41 $\theta$ and 41 $\dot{\theta}$) and 41 different actions. So 68921 different state-action combinations. It's possible to get the agent working just fine with considerably less state-value combinations and simulations. These plots just look nicer.\\

		To train the agent I first ran 40000, 1 second simulations with $\epsilon=0.7$, $\gamma=0.7$ and constant $\alpha=0.9$. On the second batch of 5000 cases I increased $\epsilon$ to $0.9$. On the last batch I increased $\epsilon$ to $1.0$ and decreased $\alpha$ to $0.0$ in order to see the ideal policy of the agent without any learning.\\

		Figure \ref{fig:ideal} shows the ideal action for each state. As it can be seen the agent is showing the expected behaviour. However it doesn't seem like it's learning properly when $\dot{\theta}\approx0$. 
		\begin{itemize}
			\item It applies a positive torque when $\theta$ is negative and vice-versa.
			\item It chooses a lower torque if the pole is already moving towards the center and increases the torque if it's moving away.
		\end{itemize}

		% The total reward sum and learning rate are low for the first few iterations as shown in Figures \ref{fig:learn} and \ref{fig:reward}. But as the agent learns more it's able to stay alive for longer, so it's able to reach more states, thus increasing the learning rate and reward sum.

		% At some point the learning rate peaks and decreases indicating that it does not have much more to learn. Both the total reward sum and learning rate settle to a "constant" value after about 15000 iterations.
